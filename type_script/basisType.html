    基础类型

    # 基础类型

    在 TS 中，数据类型可以同样分为基本类型与引用类型，除此之外还加入个别特殊类型，本节将从最基础最基础的基本类型说起：

    • 基本类型：string、number、boolean、null、undefined、symbol、bigint
    • 特殊类型：any、unknow

    基本类型

    // str 的类型设置为string，在使用过程过程中 str 的值只能是字符串，否则报错。
let str: string;
str = 'hello ts';  // Good!
str = 2;  // Error！不能将类型“number”分配给类型“string”。

let num: number = 1
let bool: boolean = true
let nul: null = null
let und: undefined = undefined
let sym: symbol = Symbol()
let big: bigint = 10n

    需要注意的是，null、undefined 类型对应的值也只有 null 与 undefined，这很好理解。

    特殊类型

    • # any

    在 TS 中，任何类型都可以归于 any 类型，并且 any 类型可以赋值给任意类型。所以 any 类型也就成了所有类型的顶级类型，同时，如果不指定变量的类型，则默认为 any 类型，当然不推荐使用该类型，因为这样丧失了 TS 的作用。

    let ay;  // 此时变量ay没有定义任何类型，且没赋值，则默认为any类型

ay = 1;  // Good!
ay = 'a';  // Good!
ay = true;  // Good!
ay = {};  // Good!
// ...

// 同样也可以显式的指定any类型
let ay2: any;

    • # unknow

    unknown 实际上就是一个类型安全的 any。作用与 any 类似，可以将任何类型的值赋给 unknown 类型。不同的是，它比 any 更加严格：

    1. unknown 类型的值不能随意赋值给其他类型，它只能赋值给 unknown 与 any 类型。
    2. 使用 unknown 类型定义的值，不能对值进行任何操作（如：方法、new 等），但是 any 可以。
    let unk: unknown;
let an: any = [];

unk = 'a';  // Good!  
unk = 1;  // Good!
unk = true;  // Good!
unk = [];  // Good!

let str: string = an;  // Good!
    str = unk;  // Error! 不能将类型“unknown”分配给类型“string”。

an.push(1);  // Good!
unk.push(1);  // Error! “unk”的类型为“未知”。

    # 联合类型

    联合类型，表示取值可以为多种类型中的一种。该语法是使用 | 号分隔每个类型。

    let union: string | number;
union = 'TypeScript';  // Good!
union = 666;  // Good!
union = true;  // Error!

    上面代码中，变量 union 的类型就可称之为是 联合类型，该变量的值可为 string 或 number，若是其它类型的话，将会报错。

    # 类型推论

    在以上的代码中，我们声明变量的同时都给变量定义了特定的类型。其实并非每次都需要这么麻烦，在 TS 中它可以根据你的赋值推论出该变量是属于什么类型。如果未定义类型，且未赋值，则会推论为 any 类型。

    let a;  // 此时该变量已被推论为 any 类型。
a = 666;  // Good!
a = 'coderChan';  // Good!
a = true;  // Good!
// ...


let str = 'hello Typescript';  // 此时推论为 string 类型
str = 'Bye Typescript';  // Good!
str = 666;  // Error! 不能将类型“number”分配给类型“string”。
