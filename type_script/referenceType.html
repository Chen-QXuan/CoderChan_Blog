    引用类型

    # 引用类型

    • 引用类型：array、 Tuple（元组）、 object（包含 Object 和 { } ）、function

    • 特殊类型：void、never

    • # array

    在 TypeScript 中，数组类型有两种定义方式：

    1. 数据类型 + [ ]
    2. Array<数据类型>
    let ary1: number[] = [1, 2, 3];
let ary2: Array<string> = ['a', 'b', 'c'];
ary2 = ['a', 'b', 1];  // Error！不能将类型“number”分配给类型“string”。

    • # Tuple（元组）

    Tuple 也是定义数组类型的一种方式，其作用就是限制 元素类型 并且 限制个数 的数组。 Tuple 的概念存在于 TS，在 JS 上是不存在的。
    注意： 在 TS 中，是允许对 Tuple 进行扩增的（即允许使用 push 方法），但在访问上是不允许的。

    let tup: [number, string];

tup = [1, 'a'];  // Good!
tup = ['a', 1];  // Error!
tup = [1, 'a', 2];  // Error!

tup.push('b');
console.log(tup);  // [1, 'a', 'b']
console.log(tup[0]);  // 1
console.log(tup[1]);  // 'a'
console.log(tup[2]);  // Error!

    • # object

    使用 object 定义的数据类型，操作时只能修改整个对象，而不能够只修改对象里的某个属性。

    let obj: object = {a: 1};
obj.a = 2;  // Error! 类型“object”上不存在属性“a”。
obj = {name: 'coderChan', age: 22};  // Good!

    解决上面代码的问题也很简单。既然对象里不存在属性，那么只需给对象内部也定义属性就完事了。这里需要使用到 {} 来定义数据类型。

    let obj: { name: string, age: number } = { name: 'coderChan', age: 22 };

obj.name = 'Mr.Chan';  // Good!
obj.name = 123;  // Error! 不能将类型“number”分配给类型“string”。
obj.height = 180;  // Error! 类型“{ name: string; age: number; }”上不存在属性“height”。

obj = {name: 'Lily'};  // Error! 缺少属性 "age"。
obj = {name: 'Lily', age: 18, height: 170};  // Error!

    以上代码就是使用 {} 来定义对象。对象中属性的个数、类型 都应该与定义时相对应，否则 TS 将会给你报错。是不是很像 Tuple 的特性。

    # 对象的可选属性

    添加可选参数的语法很简单，只需要在可选属性名后面添加 ？ 号即可。并且可选参数可为多个。

    let obj: {name: string, age?: number};
obj = {name: 'coderChan'};  // Good!
obj = {name: 'Lily', age: 22};  // Good!

    有些时候，我们只需要为对象固定某些属性就行，其他属性都是不做限制的。这时我们可以为对象添加 任意类型属性：

    let obj: {name: string, [propName: string]: any};
obj = {name: 'coderChan'};  // Good!
obj = {name: 'Lily', age: 22, single: true};  // Good!

    以上代码中，属性名 propName 并非固定写法，写啥都行。属性名是 string 类型这很好理解，对象的所有属性都是字符串类型。注意的是需要用中括号包裹起来。
    当为对象添加了任意类型属性之后，不仅 类型不受限制，属性数量也不受限制 。仅需保证 必须属性 无误即可。

    那如果，使用 {} 定义对象时，并未定义其内部属性，将会是怎样的。直接 Coding 起来！

    let obj: {};

obj = {name: 'coderChan'};  // Good!
obj = {name: 'Mr.Chan', age: 22};  // Good!
obj.name = 'Lily';  // Error! 类型“{}”上不存在属性“name”。

    这就很明显了，与使用 object 定义的数据类型一样。对象赋值随便赋，修改属性就不行。

    定义对象类型的方式除了上面两种方式之外，还有一种就是使用 大Object 来定义（即是大写的 O）。
    注意上面的是 小object。
    Object 类型就比较有趣了。除了 null 和 undefined 之外的所有 原始类型 或 非原始类型 都可以进行赋值。

    let obj: Object;

obj = 1;  // Good!
obj = 'a';  // Good!
obj = true;  // Good!
obj = {};  // Good!
obj = function() {};  // Good!
obj = Symbol();  // Good!
obj = 10n;  // Good!

obj = null;  // Error!
obj = undefined;  // Error!

    其实这是因为在 TS 中，所有的类型都是对象类型，即每一种类型都实现了 object 类型的接口，并且满足了所有 object 类型的基本特征。

    另外，Object 与 object 有一个相同点就是，如果将一个对象进行赋值，是不能够直接修改其 对象中某个属性的。

    • # function

    众所周知，定义函数的方式有两种，一种是 function，另一种是 箭头函数。所以，在 TS 中为函数定义数据类型的地方也有两处。

    没错，我想说的就是定义 参数类型 与 函数返回值类型。

    # 定义参数类型：

    function fn(name: string, age: number) {
  console.log(`I'm ${name}`)
}
fn('coderChan', 22);  // Good!
fn('coderChan');  // Error! 应有 2 个参数，但获得 1 个。
fn('coderChan', 22, true);  // Error! 应有 2 个参数，但获得 3 个。
fn('coderChan', '22');  // Error! 类型“string”的参数不能赋给类型“number”的参数。

    不难看出，函数 fn 期望接收两个参数。分别是 string 与 number 类型。所以调用函数时的实参，必须严格按照已定义的类型进行传参。它想要什么，你就只能给他什么。多了少了都不行！

    那如果想传递一些可有可无的参数嘞，如何搞？
    看着！可有可无的东西，可以通过 ？ 来声明 可选参数，并且它可以有多个。请记住，可选参数 一定要放在最后面！

    function fn(name: string, age?: number) {
  console.log(`I'm ${name}`)
}
fn('coderChan', 22);  // Good!
fn('coderChan');  // Good!
fn('coderChan', 22, true);  // Error! 应有 1-2 个参数，但获得 3 个。

function fn2(name?: string, age: number) {};  // Error! 必选参数不能位于可选参数后。

    # 定义函数返回值类型：

    函数的返回值类型除了基础类型之外，还有另外的两种特殊类型，分别是 void 与 never 类型。

    • # void

    void 像是与 any 类型相反，表示的是 没有任何类型。当一个函数没有返回值时，那么它的返回值默认就是个 void 类型。 换句话说，当一个函数咱们不需要返回值时，就可以给它定义为 void 类型。

    const myFn = (): void => {
  console.log('hello typescript');  // Good!
  return;  // Good!
  return 1;  // Error!
  return 'abc';  // Error!
  return null;  // Error!
  return undefined;  // Good!
}

    可以看到，在一个函数定义了返回值类型为 void，如果没有 return 语句的话，将可正常编译。非要加上 return 的话，要么就是直接 return，或者 return undefined。如果把其它东西 return 出来的话将会报错。
    实际上，当不写 return 语句的时候，函数本身返回的也就是 undefined。

    那么问题来了，void 也是类型里的其中一种，那是不是也可以给变量定义 void 类型呢？

    let vo: void;
vo = 1;  // Error!
vo = 'coderChan';  // Error!
vo = true;  // Error!
vo = null;  // Error!
vo = {};  // Error!
vo = undefined;  // Good!

    可以看到，可以是可以，但是它的值只能为 undefined，似乎就没什么意义的样子了。

    • # never

    never 类型表示一个函数永远不存在返回值。
    在上面我们讲到，void 类型表示的是没有返回值，实际上它只不过是返回 undefined 而已，而 never 类型连返回 undefined 都不行。

    那么要想符合这种条件呢只有两种情况：
    要么程序报错了，那么它永远没有返回值；
    要么程序进入无限死循环了，它也不会有返回值。

    const fn1 = (): never => {};  // Error!

const fn2 = (): never => {
    console.log('Never!')
};  // Error!

const fn3 = (): never => {
    while (true) {}
};  // Good

const fn4 = (): never => {
    throw new Error("error");
};  // Good

    没错吧，就算函数体内什么都没有，也是不行。只有报错或者死循环才行。因此 never 类型是用的比较少的。
    如果给变量定义 never 类型呢？
    那真的就是 Never 了。无论赋值什么都将会报错。解决方法就是将它删掉 hhhh~
