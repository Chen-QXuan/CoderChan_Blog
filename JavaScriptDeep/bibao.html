    # 代码块

    在 JavaScript 中，只要被大括号 { } 包裹起来的范围，就是一个 块。其作用主要是划分作用域，使不同的 块 避免相互影响导致污染。

    如果在代码块 {...} 内声明一个变量，那么这个变量只在该代码块内可见：

    {
    let str = 'hello';   // 仅在此代码块内可见
    console.log(str);  // 'hello'
}

console.log(str);  // Error!

    在上面的示例中，这对大括号可以是 if、function、for... 等等，只要是在该代码块内声明的东西，就只能是它自己才可以访问。

    利用这个特性，我们可以使用它来隔离一段代码，该段代码执行自己的任务，并使用仅属于自己的变量：

    {
    let str = 'JavaScript';
    console.log(str);   // 'JavaScript'
}

{
    let str = 'Vue';
    console.log(str);   // 'Vue'
}

console.log(str);  // Error!

    如果此示例里没有用大括号将各自的作用域划分开来，则会触碰到 let 关键字的底线：禁止重复声明！

    let str = 'JavaScript';
console.log(str);   // 'JavaScript'

let str = 'Vue';   // Error!
console.log(str);

    注意

    有一种情况比较特殊：

    for(let i = 0; i < 3; i++) {
    console.log(i);   // 0，然后是 1，最后是 2
}
console.log(i);   // Error!

    从视觉上看，声明 i 的位置并不被大括号 { } 所包裹，但是形如 for 这样的结构会比较特殊：在其中声明的变量也会被视为 块 的一部分。

    # 词法环境

    “词法环境” 是一个规范对象（specification object）：它只存在于 语言规范 (opens new window) 的 “理论” 层面，用于描述事物是如何工作的。因此，我们无法在代码中获取该对象并直接对其进行操作。

    词法环境对象由两部分组成：

    1、环境记录：将自身作用域内声明的 变量、函数 等信息，作为自身属性的一个对象。

    2、对外部词法环境的引用，与外部代码相关联。

    为了使内容更清晰，接下来将分步骤对词法环境进行解剖。

    # Step 1. 变量

    一个变量只是 环境记录 这个特殊的内部对象的一个属性，“获取或修改变量” 意味着 “获取或修改词法环境的一个属性”。

    每个脚本文件都会有一个 全局 词法环境，正如上图的例子，因为 str 属于全局变量，所以也被收录至全局词法环境中。箭头表示外部引用。因为全局词法环境没有外部引用，所以箭头指向了 null。
