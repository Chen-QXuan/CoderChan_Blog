    Vite 打包后访问显示空白页的问题

    # Vite 打包后访问显示空白页的问题

    技术栈： Vue3、Vite、Nginx；

    场景： vue-router 使用的是 createWebHistory 模式，在 build 打包项目并部署到服务器上时，页面访问空白，打开控制台也并没有报错；

    首先，当页面访问出现空白页时，应当先打开控制台，查看是否出现报错的情况：

    • # 页面无法渲染，报错 404

    这种情况，往往是因为路径匹配错误。需要修改的文件如下：

    // vite.config.ts
export default defineConfig({
  base: '/lnui/oumu/Craft/',     // 修改 base 属性
})

    // router/index.ts
const router = createRouter({
  history: createWebHistory('/lnui/oumu/Craft/'),   // 为 createWebHistory 路由模式传递 base 参数
  routes,
})

    解释一下，base 所填的路径来自哪里呢？ 以我的项目举例：

    我的项目首页路径是 127.0.0.1/lnui/oumu/main.html

    而打包出来的 dist 文件夹里面的内容，被我放在了这个目录之下，因此 base 的路径就是 /lnui/oumu/Craft/

    此时，我们是可以通过 127.0.0.1/lnui/oumu/Craft/Home 去访问 Vue 的首页了。

    注意、注意、注意： 如果去访问 127.0.0.1/lnui/oumu/Craft/index.html 是会出现空白页的。因为访问的应该是路由，而不是 index.html ！！！

    • # 页面正常显示，F5 刷新页面时 报错404

      出现这种症状呢，去访问 127.0.0.1/lnui/oumu/Craft/Home 是报错 404 的，只有访问 127.0.0.1/lnui/oumu/Craft 让它自动跳转至 127.0.0.1/lnui/oumu/Craft/Home 才会出现页面。

      当自动为您跳转渲染出页面后，此时去刷新页面，自然是刷新了 127.0.0.1/lnui/oumu/Craft/Home 路径，也就是说，这个时候刷新 就等同于 直接去访问 127.0.0.1/lnui/oumu/Craft/Home 一样，所以报错 404 。

      所以就需要配置一下 Nginx 来解决这个问题：

      location /lnui/oumu/Craft {
	root html;
	index index.html
	try_files $uri $uri/ /lnui/oumu/Craft/index.html;
}

      在配置里的第一行出现的 /lnui/oumu/Craft，只要按照刚刚在 vite.config.ts 里面配置的 base 属性写上就行。

      第四行最后面的 /lnui/oumu/Craft/index.html 也是一样，前面写上路径，后面拼上 index.html。

      到这里应该是完美解决了，页面正常渲染，想怎么刷就怎么刷。

    • # 其他可能情况

      // router/index.ts
{
    path: '/home',
    // component: import('@/pages/Home/index.vue'),       // 引发空白页
    component: () => import('@/pages/Home/index.vue'),    // 正确写法
 },

      以上情况也会导致访问页面是空白页，注意需要使用箭头函数！
