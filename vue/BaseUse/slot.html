    插槽

    # 插槽

    插槽又分为 默认插槽 与 具名插槽 ，其作用概括起来就一句话：为子组件传递模板内容进行渲染。

    先拿 默认插槽 举个例子：

    <!-- Parent.vue -->
<template>
    <Children>
        hello,slot!  <!-- 插槽内容 -->
    </Children>
<template>

    在上面的代码中，我们将一段字符串传递给了子组件进行渲染。

    而 <Children> 组件的模板是这样的：

    <!-- Children.vue -->
<template>
    <div class="box">
       <slot></slot>  <!-- 插槽出口 -->
    </div>
<template>

    在这个示例中，<slot></slot> 标签代表着插槽内容的渲染位置。

    因此最终 DOM 渲染出来的结果是这样的：

    <div class="box">hello,slot!</div>

    这就是一个插槽的基本使用，当然你也可以将另一个组件当做插槽内容进行传递。只要属于 template 都可以传。

    # 具名插槽

    具体插槽就是用于 将插槽内容分别渲染到子组件不同的位置。

    举个例子：

    <!-- Children.vue -->
<div class="container">
  <header>
    <!-- 标题内容放这里 -->
  </header>
  <main>
    <!-- 主要内容放这里 -->
  </main>
  <footer>
    <!-- 底部内容放这里 -->
  </footer>
</div>

    当插槽出口有多个的情况时，<slot> 元素可以有一个特殊的属性 name，用来给各个插槽分配唯一的 标识，以确定每一处要渲染的内容：

    <!-- Children.vue -->
<div class="container">
  <header>
    <slot name="header"></slot>
  </header>
  <main>
    <slot></slot>
  </main>
  <footer>
    <slot name="footer"></slot>
  </footer>
</div>

    这类带有 name 属性的插槽被称为 具名插槽。没有提供 name 属性的 <slot> 出口会隐式地命名为 “default”。

    要为具名插槽传递不同内容时，需要使用带有 v-slot 的 <template> 元素，并将插槽出口对应的 name 值传递给 v-slot 指令：

    <!-- Parent.vue -->
<Children>
    <template v-slot:header>  <!-- v-slot 的完整写法 -->
        <h1>Here might be a page title</h1>
    </template>

    <template #default>  <!-- 这里的内容将传到 不带name属性的插槽出口位置 -->
        <p>A paragraph for the main content.</p>
        <p>And another one.</p>
    </template>

    <template #footer>  <!-- v-slot 的简写形式 -->
        <p>Here's some contact info</p>
    </template>
</Children>

    在上面的示例中，分别展示了 v-slot 的完整写法与简写形式。v-slot 的简写为 #。因此，<template #footer> 等同于 <template v-slot:footer>。

    注意：所有位于顶级的非 <template> 节点都被隐式地视为默认插槽的内容！！！

    换做大白话讲就是：所有不被 <template> 标签包裹的内容，都将被传递至 默认插槽出口。

    因此，<template #default> 将可以省略。

    # 动态插槽名

    <!-- Parent.vue -->
<Children>
  <template v-slot:[dynamicSlotName]>
    ...
  </template>

  <!-- 缩写为 -->
  <template #[dynamicSlotName]>
    ...
  </template>
</Children>

    注意动态参数的 语法限制 ！！！

    # 默认内容

    <slot></slot> 标签之间所展示的内容即是默认内容，当父组件未提供任何插槽内容时，子组件将渲染默认内容

    <!-- Parent.vue -->
<template>
    <Children />
<template>
    
    
<!-- Children.vue -->
<template>
    <div class="box">
       <slot>我是默认内容</slot>
    </div>
<template>

    此时 DOM 渲染出来的结果是这样的：

    <div class="box">我是默认内容</div>

    # 渲染作用域

    插槽内容可以访问父组件的数据作用域，因为插槽内容本身就是在父组件模板当中定义的：

    <!-- Parent.vue -->
<template>
	<div>{{ count }}</div>
    <Children>{{ count }}</Children>
<template>

    上面的示例中，两个插值 count 获取到的值都是一样的。

    但是！默认情况下，插槽内容 无法访问 子组件中的数据 ！！！

    因为当父组件想获取到子组件的数据，就涉及到了跨组件。跨组件必须 通信 才能建立连接。因此在 Vue 中也为插槽提供了相应的 props 语法。

    # 插槽 props —— 作用域插槽

    插槽在接收 props 时，也是使用 v-slot 指令，不过它将会接收到一个插槽 props对象。

    同样的，默认插槽与具名插槽也是有细微之分：

    <!-- Children.vue -->
<template>
    <div class="box">
       <slot :nums="10" str="传递"></slot>
    </div>
<template>
    
    
<!-- Parent.vue -->
<template>
    
    <Children v-slot="slotProps">
        {{ slotProps.nums }} {{ slotProps.str }}
    </Children>
    
    <!-- 等价于 -->
    
    <Children>
        <template #default="slotProps">
        	{{ slotProps.nums }} {{ slotProps.str }}
		</template>
    </Children>

<template>

    以上就是一个默认插槽接收 props 的示例：在父组件里，通过 v-slot 指令 获取到子组件传递过来的 props 对象（slotProps），此时的 slotProps 长这样： { "nums": 10, "str": "传递" }。因此也可采用 对象解构 进行获取。

    具名插槽其实也只是在 v-slot 的写法上接收个对象而已：

    <!-- Parent.vue -->
<Children>
    <template v-slot:header="headerProps">  <!-- 完整写法 -->
        {{ headerProps }}
    </template>

    <template #footer="footerProps">  <!-- 简写形式 -->
        {{ footerProps }}
    </template>
</Children>

    **需要注意的是： **

    • name 是 Vue 特别保留的属性，因此不会作为 props 传递 。

    • 当 默认插槽 与 具名插槽 同时存在时，默认插槽的props 必须在 <template> 标签上接收，不可以在组件标签上接收。

    例如：

    <!-- Parent.vue  编译报错 -->
<Children v-slot="data">  
    <div>{{ data }}</div>
    <template v-slot:header>
        {{ data }}     <!-- data 属于默认插槽，此处不可用 -->
    </template>
</Children>

    # 总结：

    • <slot></slot> 标签位置在哪里，插槽内容就渲染到哪里。标签内部包裹的内容，即是 默认内容
    • 具名插槽出口使用 name标识， 插槽内容使用 v-slot 指令搭配 <template> 标签
    • 默认情况下，插槽内容 无法访问 子组件中的数据。访问需使用插槽 props，将 v-slot 接收对象即可
    • 不同作用域的 插槽 props 不可相互访问，当 默认插槽 与 具名插槽同时存在时，默认插槽也应当使用 <template> 包裹，并且 默认插槽的 props 不可在组件标签上接收
