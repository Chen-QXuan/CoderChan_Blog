    # v-model

    # 组件 v-model

    回忆一下，在原生元素上 v-model 的用法：

    <input type="text" v-model="searchText" >

    然而，模板编译器会对 v-model 进行更冗长的等价展开。因此上面的代码其实等价于下面这段：

    <input type="text" :value="searchText" @input="searchText = $event.target.value">

    而如果我们将 v-model 写在一个组件上时，将会是这样：

    <Child v-model="searchText" />

 <!-- 展开等价于以下 -->

<Child :modelValue="searchText" @update:modelValue="newValue => searchText = newValue" />

    注意：modelValue 和 update:modelValue 都是组件 v-model 上 默认传递的 prop 和 自定义事件

    既然是 prop 与 自定义事件，那么为了让这段代码能够顺利工作起来，组件内部应该做两件事：

    1. 接收 一个 modelValue 的 prop，并绑定到 <input> 的 value 属性上；
    2. 监听 <input> 输入框的变化情况，将新值作为参数，并 触发 update:modelValue 自定义事件。

    以上两点需求，用代码的形式翻译出来又有两种方式：

    • 第一种：
    <!-- Child.vue -->
<script setup>
  defineProps(['modelValue'])
  defineEmits(['update:modelValue'])
</script>

<template>
  <input :value="modelValue" @input="$emit('update:modelValue', $event.target.value)" />
</template>
    • 第二种：
    <!-- Child.vue -->
<script setup>
  import { computed } from 'vue';
  const props = defineProps(['modelValue'])
  const emit = defineEmits(['update:modelValue'])
  
  const searchValue = computed({
    get() {
      return props.modelValue
    },
    set(newValue) {
      emit('update:modelValue', newValue)
    }
  })
</script>

<template>
  <input v-model="searchValue" />
</template>

    以上两种方法，都能使 v-model 在组件上正常运行了：

    <Child v-model="searchText" />

    回到上面所说过的：modelValue 和 update:modelValue 都是组件 v-model 上 默认传递的 prop 和 自定义事件。

    既然是默认的，那么就肯定有办法去修改它。

    这个时候就需要用到 v-model 的参数 了。

    # v-model 的参数

    默认情况下，v-model 在组件上都是使用 modelValue 作为 prop，并以 update:modelValue 作为对应的事件。但是我们可以通过给 v-model 指定一个参数来更改这些名字：

    方式：v-model:xxx —— 在 v-model 后面加上冒号，并写上你的参数

    <Child v-model:title="searchText" />

<!-- 展开等价于以下 -->

<Child :title="searchText" @update:title="newValue => searchText = newValue" />

    组件内部：

    <!-- Child.vue -->
<script setup>
  defineProps(['title'])
  defineEmits(['update:title'])
</script>

<template>
  <input :value="title" @input="$emit('update:title', $event.target.value)" />
</template>

    可以看到，组件内部只是将 modelValue 替换为任意的参数而已，不变的是，自定义事件依然需要 update。

    computed 方式同理，这里不再赘述。

    # 多个 v-model 绑定

    在上面的介绍当中，组件内部的 <input> 都是只有一个而已，那么当有多个的情况是应该如何解决呢？

    答：引用组件时，组件上 写多几个 v-model 不就好了

    当然，正常情况下，多个 v-model 必须搭配 v-model 参数去玩（特殊情况除外...）

    <!-- Parent.vue -->
<template>
  <Child v-model:name="bookName" v-model:price="bookPrice" />
</template>

<!-- Child.vue -->
<script setup>
  defineProps(['name', 'price'])
  defineEmits(['update:name', 'update:price'])
</script>

<template>
  <input :value="name" @input="$emit('update:name', $event.target.value)" />
  <input :value="price" @input="$emit('update:price', $event.target.value)" />
</template>

    # v-model 修饰符

    我们知道 v-model 有一些内置的修饰符 (opens new window)，例如 .trim，.number 和 .lazy。不过，我们也可以自定义一个 v-model 修饰符：

    方法：v-model.xxx —— 只需在 v-model 后面加个点，并给它起个名字

    举个例子，我们想要自定义个修饰符，这个修饰符的作用会帮我们把输入的字符串，第一个字母转为大写：

    <!-- Parent.vue -->
<template>
  <Child v-model.capitalize="myText" />
</template>

    注意：组件的 v-model 上所添加的修饰符，可以通过 modelModifiers prop 在组件内部访问，该 prop 也是默认传递的，是一个对象。

    <!-- Child.vue -->
<script setup>
  const props = defineProps({
      modelValue: String,
      modelModifiers: {
          default: {}
      }
  })
  
  console.log(props.modelModifiers);   // { capitalize: true }
  
  const emit = defineEmits(['update:modelValue'])
  
  const changeValue = e => {
      let value = e.target.value
      if(props.modelModifiers.capitalize) {
          value = value.charAt(0).toUpperCase() + value.slice(1)
      }
      emit('update:modelValue', value)
  }
</script>

<template>
  <input :value="modelValue" @input="changeValue" />
</template>

    如上，我们在组件内将默认传递的 modelModifiers prop，接收一下，之所以需要将它的默认值设为一个空对象，是因为避免在 if 语句里报错。这不难理解。

    将 modelModifiers 输出一下，可以看到 对象里面有个名为 capitalize 的属性，其值为 true，那是因为我们在引用组件时，给它添加了这么一个修饰符，因此在这里就能看到。

    值得一提的是，对于又有参数又有修饰符的 v-model 绑定，生成的 prop 名将是 参数 + "Modifiers"。并且，参数必须在修饰符的前面！！！

    <!-- Parent.vue -->
<template>
  <Child v-model:title.capitalize="myText" />
</template>

    组件内对应的声明是：

    const props = defineProps(['title', 'titleModifiers'])
defineEmits(['update:title'])

console.log(props.titleModifiers)   // { capitalize: true }

    # 总结

    • 组件 v-model 默认情况下，必须通过 modelValue prop 与 update:modelValue 自定义事件 进行绑定
    • 组件 v-model 修饰符，需要通过 modelModifiers prop 进行接收
    • 既带参数又带修饰符的组件 v-model，参数必须在修饰符的前面
