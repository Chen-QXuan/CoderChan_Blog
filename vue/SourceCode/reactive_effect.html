    reactive & effect

    # reactive & effect

    在研究 reactive 之前，需要先了解 effect 的作用。因为 reactive 需要和 effect 配合 使用才会有响应式的效果。

    不过通常情况下我们是不会直接使用 effect 的，因为 effect 是一个底层的API，在我们使用 Vue3 的时候 Vue 默认会帮我们调用 effect（当然也可以通过 import 进行使用），所以我们的关注点通常都是在reactive上。

    # 什么是 effect

    • effect 是个方法，该方法的作用是创建一个副作用函数，这个函数会在依赖的响应式数据发生变化的时候执行；

    • 参数：

      1. { Function }： 必传参数，该参数是个回调函数，调用 effect 时会自动触发一次。当回调函数里依赖的 响应式对象 发生变化时，该函数会被再次执行。
      2. { Object }：可选参数
        • { Function } scheduler：可选参数。若传递该参数时，当响应式 set => 执行该函数，而并非执行 第一参数；若未传递时，响应式 set => 执行第一参数；
        • { Function } onStop：可选参数。当调用 stop 函数时，该函数会被触发。

    • 返回值：

      effect 函数会返回一个 runner 函数，该函数本质就是 effect 的副作用函数，因此调用 runner 函数也就会执行 effect 的回调函数，自然地，该返回值也就是 effect 回调的返回值。

    # stop函数

    • stop 方法需要接收一个 runner 函数，并且该参数是必传的。
    • 调用 stop 函数之后，当响应式对象发生改变，并 不会再次执行effect(fn)中的fn函数。

    # reactive()

    在 vue3 中，reactive 函数底层是通过 Proxy 代理对象来达到响应式的效果。

    在 reactive 里面会创建一个 Proxy 对象，通过 get 方法来进行 收集依赖，set 方法来 触发依赖。最终将这个 Proxy 对象 return 出去。

    # 实现一下

    # effect 大致思路

    1. 需要创建一个 响应式依赖，用于存储 effect 的回调函数；
    2. 当调用 effect 函数时，传入的回调函数会立即执行一次；
    3. effect 会返回一个 runner 函数，该函数也就是 effect 的回调；因此 runner 的返回值正是 effect 回调的返回值。
    class ReactiveEffect {
  private _callback: Function;
    
  constructor(callback){
    // 收集 effect 回调函数
    this._callback = callback;
  }

  // run 也就是执行了收集到的回调
  // 3、需要 return 是因为 runner 函数的返回值，就是该回调的返回值。   
  run() {
    return this._callback();
  }
}

// effect 逻辑实现
function effect(fn) {
  // 1、创建一个 响应式依赖
  const dep = new ReactiveEffect(fn)
  
  // 2、当调用effect时，执行一次依赖
  dep.run()
  
  // 3、
  // 之所以需要使用bind，是因为run方法里面涉及到一个this的指向问题
  // 如果直接return dep.run; 之后再去执行runner时，this指向的便是window。因此会导致执行不到已收集的依赖。
  return dep.run.bind(dep)
}

    # reactive 实现

    function reactive(raw) {
  return new Proxy(raw, {
    get(target, key) {
      const res = Reflect.get(target, key)  // 等同于 => const res = target[key]
      
	  // 如果是对象嵌套，接着转换就行
      if(isObject(res)) {
        return reactive(res)
      }
        
      // 收集依赖
      track(target, key)
      return res
    },

    set(target, key, newVal) {
      const res = Reflect.set(target, key, newVal)  // 等同于 => const res = (target[key] = newVal)
      
      // 触发依赖
      trigger(target, key)
      return res
    }
  })
}

    # 依赖的收集、触发

    因为每个effect函数在调用时，都会创建一个 响应式依赖，并自动执行该依赖。如果说 这个依赖函数 里面读取到了 响应式对象的属性，那么将会把这个依赖给收集起来。

    换句话说，每个响应式对象的属性，都会有属于自己的依赖。

    为了让每个属性key都能找到自己的 响应式依赖，所以创建 响应式依赖 的类也应当调整一下：

    // 用于存储当前依赖
let activeDep;

class ReactiveEffect {
  // ....
    
  run() {
    // 存储当前激活的响应式依赖，用于依赖收集
    activeDep = this
    // 存储后再执行依赖，依赖中若触发到get时，才能精准收集
    return this._callback();
  }
}

    // 全局Map，存储所有的 依赖
const effectMap = new Map()

// 收集依赖
function track(target, key) {
  // 每个响应式对象都是一个Map
  let targetMap = effectMap.get(target)
  // 初始化时，该Map不存在，应手动创建
  if(!targetMap) {
    targetMap = new Map()
    effectMap.set(target, targetMap)
  }

  // 响应式对象的每个属性，则是一个 Set。因为所收集到的 响应式依赖 都不应重复。
  let keyDep = targetMap.get(key)
  // 初始化时手动创建 Set
  if(!keyDep){
    keyDep = new Set()
    targetMap.set(key, keyDep)
  }

  if(!activeDep) return 
  // 当属性被访问到时，触发的收集依赖，也正是当前被激活的 响应式依赖
  keyDep.add(activeDep)
}

// 触发依赖
function trigger(target, key) {
  // 通过响应式对象 获取到对应的 Map
  const targetMap = effectMap.get(target)
  // 通过属性key，获取到对应的依赖集合 Set
  const deps = targetMap.get(key)
  // 所有依赖于该属性key的函数，都会被执行
  deps.forEach(dep => {
      dep.run()
  });
}

    # 代码完善

    effect 完善：

    在介绍 effect 时说到，它可接收第二个参数，即对象里的 scheduler、onStop 方法；因此首先需要对刚刚手写的 effect 调整完善：

    function effect(fn, options: any = {}) {
  // 创建一个 响应式依赖
  const dep = new ReactiveEffect(fn)
  
  // 合并options对象，主要用于合并options里的方法。例如 scheduler、onStop 等
  Object.assign(dep, options)
  
  // 当调用effect时，执行一次依赖
  dep.run()
  
  // 之所以需要使用bind，是因为run方法里面涉及到一个this指向问题
  // 如果直接return dep.run; 之后再去执行runner时，this指向的便是window。因此会导致执行不到已收集的依赖。
  const runner = dep.run.bind(dep)

  // 将实例对象挂载到runner函数上，用于调用stop函数时，可以找到自己的实例对象
  runner.instanceDep = dep

  return runner
}

    trigger 完善、scheduler 执行时机：

    // 触发依赖
function trigger(target, key) {
  const targetMap = effectMap.get(target)
  const deps = targetMap.get(key)
  deps.forEach(dep => {
    // 在创建实例时，若传递了 scheduler, 在触发依赖时，应当执行 scheduler
    if(dep.scheduler) {
      dep.scheduler()
    }else if(!dep.isStop) {
      // 若执行了stop函数，将不再执行依赖
      dep.run()
    }
  });
  
}

    stop 函数：

    function stop(runner) {
  // 执行 runner函数 所属实例的 stop 函数
  runner.instanceDep.stop()
}

    响应式依赖完善、onStop 执行时机：

    class ReactiveEffect {
  private _callback: Function;
  // 标识是否已执行stop函数
  isStop = false
  scheduler?: () => void
  onStop?: () => void
  constructor(callback){
    // 收集 effect 回调函数
    this._callback = callback;
  }

  run() {
    activeDep = this
    return this._callback();
  }

  stop() {
    this.isStop = true
    // 若传递了onStop方法，则在调用 stop 方法时执行
    this.onStop && this.onStop()
  }
}
