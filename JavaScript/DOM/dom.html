    获取节点

    # 获取节点

    • # getElementBy（推荐）

    获取到的元素列表是 动态集合（会实时更新）。
    注意：此方法除了通过 id 获取之外，其他获取到的是一个 伪数组 （因 id 具有唯一性），伪数组不支持数组的大部分 api 。若要操作需遍历，也可以在后面加下标。遍历时需使用 Array.from(节点) 转换为数组

    • getElementById —— 通过 ID 获取节点
    • getElementsByClassName —— 通过类名获取节点
    • getElementsByTagName —— 通过标签名获取节点
    • getElementsByName —— 通过 name 值获取节点

    • # querySelector

    获取到的元素列表是 静态集合（不会实时更新）

    • document.querySelectorAll("选择器"); —— 获取数组类型，此方法获取到的数组可直接遍历
    • document.querySelector("选择器"); —— 只会获取到一个，若有相同也只是获取到第一个

    两者区别

    <body>
    <ul>
        <li>Lisa</li>
        <li>Jennie</li>
        <li>Rose</li>
        <li>Jisoo</li>
    </ul>
</body>
<script>

    let ul1 = document.getElementsByTagName('ul');  // HTMLCollection [ul]
    let ul2 = document.querySelectorAll('ul');  // NodeList [ul]

    let li1 = document.getElementsByTagName('li')
    let li2 = document.querySelectorAll('li')

    ul1[0].appendChild(document.createElement('li'));
    ul2[0].appendChild(document.createElement('li'));
    
    console.log(li1.length);  // 6
    console.log(li2.length);  // 4

    
    // 可以获取拥有多个类名的元素, 但是要用空格隔开
    document.getElementsByClassName("class1 class2");
    
</script>

    在上面的代码中，因为通过 getElementsByTagName 获取到的是动态集合，所以输出 6 。而静态集合 querySelectorAll 输出的则是原来的 4 。

    性能对比

    console.time('querySelectorAll');
for (let i = 0; i < 1000; i++) {
    document.querySelectorAll('li');
}
console.timeEnd('querySelectorAll');  // querySelectorAll: 0.526123046875 ms

console.time('getElementsByTagName');
for (let i = 0; i < 1000; i++) {
    document.getElementsByTagName('li');
}
console.timeEnd('getElementsByTagName');  // getElementsByTagName: 0.14501953125 ms

    可以看到，getElementsBy 方法的运行速度比 querySelector 快了好几倍；也就是说 getElementsBy 的性能确实是好很多，所以一般对 DOM 节点的获取优先使用 getElementsBy ，对于选择器比较复杂（多层嵌套的）再考虑使用 querySelector。

    注意

    以下属性中，子节点 与 子元素节点不一定相同，子节点可包括 文本节点（换行也算节点），而 子元素节点 一定是个 标签节点。也就是说直接过滤无效节点，只获取元素节点。以下属性当无对应节点时，返回 null。

    • # parentNode —— 获取元素的父节点

    节点.parentNode

    • # parnetElement —— 获取元素的父节点（new）

    • # childNodes —— 获取所有的子节点（包括文本节点和注释节点）

    • # firstChild —— 获取元素的第一个子节点

    • # firstElementChild —— 获取第一个子元素节点（new）

    • # lastChild —— 获取元素的最后一个子节点

    • # lastElementChild —— 获取最后一个子元素节点（new）

    • # nextSibling —— 获取元素的下一个同胞节点

    • # nextElementSibling —— 获取下一个元素节点（new）

    • # previousSibling —— 获取元素的上一个同胞节点

    • # previousElementSibling —— 获取上一个元素节点（new）

    # 操作节点

    • # createElement —— 创建节点

    语法： document.createElement("标签名")

    • # appendChild —— 插入节点

    语法： 父元素.appendChild(要插入的子元素)

    把一个元素插入父元素内部的结尾部分。
    注意：创建的一个节点只能插入一次。创建的节点多次插入页面中，只有最后一次插入有效！

    • # removeChild —— 删除节点

    语法： 父元素.removeChild("要删除的元素")

    删除掉的元素会从页面消失，但内存中还有。
    会返回被删除掉的节点。被删除掉的节点在内存中存在，所以可以重新使用该节点

    • # replaceChild —— 替换节点

    语法： 父元素.replaceChild("newChild", "oldChild")

    用新的节点（newChild）替换旧的节点（oldChild）

    • # cloneNode —— 克隆节点

    语法： 元素.cloneNode(true || false)

    false 只拷贝当前节点，true 拷贝当前节点和它的子节点

    • # createTextNode —— 创建文本节点

    语法： document.createTextNode("文本内容")

    文本节点也是节点，可以作为节点插入

    # 常用属性

    const div = document.getElementsByTagName('div')[0];

// 节点.id => 获取id
console.log(div.id);
// 修改id
div.id = 'myDiv';  

// 节点.style => 获取行内样式
console.log(div.style);  
// 节点.style.样式 = 样式值;  => 设置元素的行内样式
div.style.color = "red"; 
// 节点.style.样式 => 获取行内指定样式
console.log(div.style.border);  

// 节点.className => 获取类名
console.log(div.className);
// 替换掉其他类名
div.className = 'newClass';
// 添加类名。注意新类名前面需要加一个空格
div.className += ' class1 class2'

    # classList

    const div = document.getElementsByTagName('div')[0];

// 节点.classList => 获取类名列表(类名组成的伪数组)
console.log(div.classList);

// 拿到类名字符串
console.log(div.classList.value);

// add 添加一个或多个类名
div.classList.add("class","class2","class3");

// remove 删除一个或多个类名
div.classList.remove("class","class2")

// replace 修改类名 (被换类名，新类名)
div.classList.replace("class3", "newClass")

// toggle 切换类名 (有这个类名就去掉, 没有这个类名就加上)
div.classList.toggle("active")

    # 节点属性

    • 元素.nodeName —— 获取节点名称
    • 元素.nodeValue —— 获取节点的值
    • 元素.nodeType —— 获取节点类型
    节点名称 节点的值 节点类型
    元素节点 大写的元素名本身 null 1
    属性节点 属性名本身 属性值（可修改） 2
    文本节点 #text 文本内容（可修改） 3
    注释节点 #comment 注释内容（可修改） 8
    文档节点 #document null 9

    # 元素内容

    以下属性皆可用于获取与设置

    • # innerText

    获取或设置元素内部的纯文本内容，换行的文本节点不会被获取（文本不会被解析成HTML标签）。
    注意：此属性获取不到被隐藏的元素！

    • # textContent

    获取或设置元素内部的纯文本内容，包括换行的文本节点（文本不会被解析成HTML标签）。
    注意：被隐藏的子元素的文本内容依然能被获取！

    • # innerHTML

    获取或设置元素内部的所有内容，包括注释节点等等...（文本会被解析成HTML标签）。

    # 表单元素

    • # 表单元素.value —— 设置与获取表单元素的 value 属性值

    • # 表单元素.placeholder —— 设置与获取表单元素的 placeholder 属性值

    # 更新访问属性

    • # 元素.getAttributeNode("属性名") —— 获取元素的属性节点

    • # 元素.getAttribute("属性名") —— 获取元素的属性值

    • # 元素.setAttribute("属性名", "属性值字符串") —— 设置元素的属性值

    • # 元素.hasAttribute() —— 判断元素是否含有某个属性（boolean）

    • # 元素.removeAttribute("属性名") —— 移除指定属性

    # 自定义属性

    自定义属性长这样： data-属性名=“自定义属性值”

    获取自定义属性：元素.dataset.属性名 或者 元素.dataset["属性名"]
    设置自定义属性：元素.dataset.属性名 = 新值 或者 元素.dataset["属性名"] = 新值
