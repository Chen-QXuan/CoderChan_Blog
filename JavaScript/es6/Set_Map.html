    Set

    # Set

    • 定义： 类似于数组，但是成员的值都是唯一的，没有重复的值。 在 Set 集合中，不会对所存值进行强制类型转换，因此数字 5 和字符串 “5” 可以作为两个独立元素存在。引擎内部使用的是 Object.is( ) 方法检测两个值是否一致，唯一例外的是，Set 集合中的 +0 和 -0 被认为是相等的。 当然，如果向 Set 集合中添加多个对象，则它们之间彼此保持独立（它们引用着不同的地址）。

    • 声明： const set = new Set(arr)

    • 参数： 具有 Iterator 接口的数据结构

    • 属性：

      • size： 返回 Set 对象中值的个数

    • 方法：

      • add( )： 在 Set 对象尾部添加一个元素。返回该 Set 对象
      • delete( )： 移除 Set 中与这个值相等的元素，有则返回 true，无则返回 false
      • clear( )： 清除 Set 的所有元素
      • has( )： 是否存在这个值，如果存在为 true，否则为 false
    // 调用 new Set() 创建 Set 集合
const set = new Set();

// add()
set.add(5);
set.add('5');
set.add(5);  // Set方法具有非重复性，因此此次添加将被忽略
console.log(set);  // Set(2) {5, '5'}

// size
console.log( set.size );  // 2

// delete()
set.delete('5');  // true
console.log(set);  // Set(1) {5}
console.log( set.size );  // 1

// clear()
set.clear();
console.log(set);  // Set(0) {size: 0}

// has()
console.log( set.has(5) );  // false
    • 遍历操作：
      • keys( )： 返回键名的遍历器
      • values( )： 返回键值的遍历器
      • entries( )： 返回键值对的遍历器
      • forEach( )： 遍历每个元素

    keys( )，values( )，entries( ) 方法返回的都是遍历器对象。由于 Set 结构没有键名，只有键值（或者说键名和键值是同一个值），所以 keys 方法和 values 方法的行为完全一致 。

    const set = new Set(['red', 'green', 'blue', 'red'])

// keys()
for(let item of set.keys()) {
  console.log(item);
};  // red   // green   // blue

// values()
for(let item of set.values()) {
  console.log(item);
};  // red   // green   // blue

// entries()
for(let item of set.entries()) {
  console.log(item);
};    // ['red', 'red']   // ['green', 'green']   // ['blue', 'blue']

// 注意：Set 结构的实例默认可遍历，它的默认遍历器生成函数就是它的values方法。
// 这意味着，可以省略values方法，直接用 for...of 循环遍历 Set ！！！
for(let item of set) {
  console.log(item);
};  // red   // green   // blue

    Set 集合的 forEach 或许并不是你想象的那样，它与数组中的 forEach 有一个奇怪的差别，因此拎出来单独介绍。

    forEach( ) 方法的回调函数接受以下 3 个参数：

    • Set 集合中下一次索引的位置
    • 与第一个参数一样的值
    • 被遍历的 Set 集合本身

    What ？前两个参数的值竟然是一样的 ？！
    是的，你没看错。就是一样！尽管这看起来像是一个错误，但其实也解释的通。

    数组和 Map 集合的 forEach( ) 方法的回调函数都接受 3 个参数，前两个分别是 值 和 键名（对于数组来说就是数值型索引值）。
    所以不要忘了前面刚说啥来着，Set 集合没有键名！！！ 因此他们是一样的值。

    除了回调函数参数稍有不同之外，其他完全相同，依然是你心目中的那个它。

    const set = new Set(['coder', 'Chan'])

set.forEach((value, key) => console.log( key + ' - ' + value ))
// coder - coder
// Chan - Chan

    • 将 Set 集合转换为数组

      将数组转换为 Set 集合的过程很简单，只需给 Set 构造函数传入数组即可；将 Set 集合再转回数组的过程更简单，只需使用 展开运算符（ ... ），它可以将数组中的元素分解为各自独立的函数参数。

    // 将数组转换为 Set 对象
const set = new Set([1, 2, 3, 3, 3, 4, 5]);

// 转回数组
const ary = [...set]
console.log(ary);  // [1, 2, 3, 4, 5]

    # WeakSet

    问号三连开局：
    WeakSet 是啥玩意？为啥会有这东西？它与 Set 的区别又是啥？

    请看以下分解 ！

    将对象存储在 Set 的实例与存储在变量中完全一样，只要 Set 实例中的引用存在，垃圾回收机制就不能释放该对象的内存空间，于是 Set类型可以被看作是一个强引用的 Set 集合。

    拿个🌰给你看：

    let set = new Set(),
    key = {};
set.add(key);

console.log(set.size);  // 1
console.log(set);  // Set(1) {{…}}

// 移除原始引用
key = null;
console.log(set.size);  // 1
console.log(set);  // Set(1) {{…}}

// 重新取回原始引用
console.log([...set][0]);  // {}

    在上面这颗栗子中，将变量 key 设置为 null 时便清除了对初始对象的引用，但是 Set 集合却保留了这个引用，你仍然可以使用展开运算符将 Set 集合转换成数组格式并从数组的首个元素取出该引用。大部分情况下这段代码运行良好，但有时候你会希望当其他所有引用都不再存在时，让 Set 集合中的这些引用随之消失。

    为了解决这个问题，ES6 老板 请来了另外一个类型：WeakSet 集合（ 弱引用 Set 集合 ）。WeakSet 集合只存储对象的弱引用，并且不可以存储原始值；集合中的弱引用如果是对象唯一的引用，则会被回收并释放相应内存。

    • 声明： const set = new WeakSet(obj)
    • 参数： 具有 Iterator 接口的数据结构
    • 方法：
      • add( )： 在 Set 对象尾部添加一个元素。返回该 Set 对象
      • has( )： 是否存在这个值，如果存在为 true，否则为 false
      • delete( )： 移除 WeakSet 的中与这个参数相等的引用，有则返回 true，无则返回 false
    // 调用 new WeakSet() 创建 WeakSet 集合
const set = new WeakSet();

const key = {};

// add()
set.add(key);
console.log(set);  // WeakSet {{…}}

// has()
console.log( set.has(key) );  // true
console.log( set.has({}) );  // false

// delete()
set.delete(key);  // true
set.delete({});  // false
console.log(set);  // WeakSet {}

// WeakSet 没有 size 属性！！！
console.log(set.size);  // undefined

    • 两种 Set 类型的主要区别

      两种 Set 类型之间最大的区别是 WeakSet 保存的是对象值的弱引用

    let set = new WeakSet(),
    key = {};

set.add(key);
console.log(set.has(key));

// 移除原始引用
key = null;

console.log(set); 
console.log(set.has(key));

    这段代码执行过后，已经移除了对象 key 的最后一个强引用了，也就无法访问 WeakSet 中 key 的引用了。此时 JavaScript 引擎也移除了最后一个弱引用。

    • 其他差异：

      • 在 WeakSet 的实例中，如果向 add( ) 方法传入非对象参数会导致程序报错，而向 has( ) 和 delete( ) 方法传入非对象参数则会返回 false。
      • WeakSet 集合不可迭代，因此不能被用于 for...of 循环。
      • WeakSet 集合不暴露任何迭代器（例如 keys( ) 和 values( ) 方法），所以无法通过程序本身来检测其中的内容。
      • WeakSet 集合不支持 forEach( ) 方法。
      • WeakSet 集合不支持 size 属性。

    • 总结：

      如果只需要跟踪对象引用，那么更应该使用 WeakSet 集合而不是普通的 Set 集合。

    # Map

    JavaScript 的对象（Object），本质上是键值对的集合（Hash 结构），但是传统上只能用字符串当作键。这给它的使用带来了很大的限制。为了解决这个问题，ES6 老板请来了 Map 数据结构。

    • 定义： Map 类型类似于对象，是一种储存着许多 键值对 的有序列表，其中的键名和对应的值支持 任何类型 的。键名的等价性判断是通过调用 Object.is( ) 方法实现的。所以数字 5 与字符串 “5” 会被判定为两种类型，可以分别作为独立的两个键出现在程序中，这一点与对象中不太一样，因为对象的属性名总会被强制转换成字符串类型。也就是说，Object 结构提供了 “字符串：值” 的对应，Map 结构提供了 “值：值” 的对应，是一种更完善的 Hash 结构实现。如果你需要 “键值对” 的数据结构，Map 比 Object 更合适。

    • 声明： const map = new Map(arr)

    • 参数： 可以向 Map 构造函数传入数组来初始化一个 Map 集合。数组中的每个元素都是一个子数组，子数组中包含一个键值对的 键名 与 值 两个元素。因此 Map 集合的参数将是个二维数组。

    • 属性：

      • size： 返回 Map 集合中键值对的数量

    • 方法：

      • set( )： 向 Map 集合添加新的元素。返回该 Map 对象
      • get( )： 返回该 键名 对应的 值
      • delete( )： 从 Map 集合中移除指定键名及其对应的值，有则返回 true，无则返回 false
      • clear( )： 清除 Map 中的所有键值对
      • has( )： 检测指定的键名在 Map 集合中是否存在，如果存在为 true，否则为 false
    // 调用 new Map() 创建 Map 集合
const map = new Map();

// set()
map.set('title', 'Weblog');
map.set('year', 2022);

console.log(map);  // Map(2) {'title' => 'Weblog', 'year' => 2022}

// size
map.size;  // 2

// get()
map.get('year');  // 2022

// delete()
map.delete('year');  // true
console.log(map);  // Map(1) {'title' => 'Weblog'}

// has()
map.has('year');  //false
map.has('title');  // true

// clear()
map.clear();
map.has('title');  // false
map.get('title');  // undefined
map.size;  // 0
    • 遍历操作：
      • keys( )： 返回键名的遍历器
      • values( )： 返回键值的遍历器
      • entries( )： 返回键值对的遍历器
      • forEach( )： 遍历每个元素

    需要特别注意的是，Map 的遍历顺序就是插入顺序。

    const map = new Map([['name', 'coderChan'], ['height', 180]])
map.set('age', 22)

// keys()
for(let key of map.keys()) {
  console.log(key);
};  // 'name'   // 'height'   // 'age'

// values()
for(let value of map.values()) {
  console.log(value);
};  // 'coderChan'   // 180   // 22

// entries()
for(let item of map.entries()) {
  console.log(item);
};    // ['name', 'coderChan']   // ['height', 180]   // ['age', 22]

// Map 结构的默认遍历器接口（Symbol.iterator属性），就是entries方法。
for(let [key, value] of map) {
  console.log(key + '-' + 'value');
};    // 'name-value'   // 'height-180'   // 'age-22'
map[Symbol.iterator] === map.entries;  // true

// forEach( ) 还是你心目中的那个它
map.forEach((value, key) => {
  console.log(key + ':' + value)
});    // name:coderChan   // height:180   // age:22
    • Map 与其他数据结构的互相转换：
    // 将Map转为数组最方便的方法，就是使用扩展运算符（...）
const map = new Map([['name', 'coderChan'], ['height', 180]])
[...map];   // [['name', 'coderChan'], ['height', 180]]


// 将数组直接传入 Map 构造函数，就可以转为 Map
// 注意是二维数组，且是两元素数组
const ary = [['name', 'coderChan'],[{hobby: 'coding'}, 'favourite']];
new Map(ary);  // Map(2) {'name' => 'coderChan', {hobby: 'coding'} => 'favourite'}


// Map 转为对象 如果所有 Map 的键都是字符串，它可以无损地转为对象
function strMapToObj(map) {
  const obj = {};
  for(let [key, value] of map) {
    obj[key] = value
  }
  return obj
};
const map = new Map([['name', 'coderChan'], ['hobby', 'coding']]);
strMapToObj(map);  // {name: 'coderChan', hobby: 'coding'}


// 对象转为 Map 可以通过 Object.entries()
const obj = {name: 'coderChan', hobby: 'coding'};
new Map(Object.entries(obj));  // Map(2) {'name' => 'coderChan', 'hobby' => 'coding'}
